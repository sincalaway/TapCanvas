Date: 2026-04-11

## Purpose

这不是工程实现 checklist，而是给产品负责人、运营、测试和新用户直接使用的“上手路径 checklist”。

目标：

- 看完就知道现在系统已经做成什么样
- 不需要理解代码，也能一步步把一个漫剧项目走通
- Web 端必须有与这些步骤对应的暗示、入口和提示文案

## Product Position

TapCanvas-pro 当前不是单纯的“画布工具”。

它现在的产品定位应该理解成：

- `project` = 作品空间 / 共享资源容器
- `chapter` = 默认生产工作台 / 导演台
- `shot` = 最小执行和审阅单元
- `studio flow` = 附着在 project/chapter/shot 上的执行链
- `workspace` = 生产总控台

## User-Facing Main Route

### Route A: First-time user with no project

1. 打开 `/`
2. 系统进入项目管理页 `/projects`
3. 用户创建第一个项目
4. 进入章节型工作流

Web hint requirements:

- 项目管理页必须有清晰的“创建第一个项目”入口
- 空状态必须告诉用户支持：
  - 空白项目
  - 文本上传项目
  - 先建项目、后补文本

### Route B: Existing user with project

1. 打开 `/`
2. 系统进入 `/workspace`
3. 用户看到最近章节、返工镜头、运行中任务
4. 点击“继续最近章节”
5. 进入当前章节工作台

Web hint requirements:

- `workspace` 顶部必须有一张“继续生产”主卡
- 必须展示本轮最推荐动作，而不是只展示指标
- 如果有项目但没有最近章节，也必须给出下一步建议

### Route C: Chapter production route

1. 用户进入某一章
2. 检查这章是否已经绑定原文章节
3. 如果未绑定：
   - 手动绑定本章
   - 或点击智能绑定本章
   - 或点击自动绑定未映射章节
4. 查看章节文本窗口
5. 创建或整理镜头
6. 选择当前镜头
7. 生成镜头概念图
8. 选择最佳结果
9. 提升为场景物料
10. 查看资源影响面

Web hint requirements:

- 章节页必须显式告诉用户“当前最该做什么”
- 未绑定原文时，必须优先提示绑定动作
- 已绑定但无镜头时，必须提示新建镜头
- 已有镜头但未出图时，必须提示生成镜头
- 已有资产时，必须提示查看影响面和审阅动作

### Route D: Shot review route

1. 选择镜头
2. 检查镜头标题、摘要、状态
3. 查看当前结果图
4. 结果可用则点击“审核通过”
5. 结果不可用则点击“打回返工”
6. 必要时重新生成
7. 如果镜头依赖共享资产，查看影响面是否过期

Web hint requirements:

- 镜头编辑区必须有清晰的“审核通过 / 打回返工”动作
- 不能让用户只靠改状态下拉框完成审阅
- 当前镜头的资源信息必须和镜头审阅区域紧邻

### Route E: Flow route

1. 在章节页点击“进入章节流程”
2. 或在镜头页点击“进入镜头流程”
3. 进入 `/studio`
4. 顶部看到当前宿主：
   - 项目流程
   - 章节流程
   - 镜头流程
5. 保存时写回对应宿主，不污染其他层

Web hint requirements:

- Studio 顶部必须显示当前宿主 badge
- 用户必须知道自己编辑的是 project/chapter/shot 哪一层

## Operational Checklist

### 01. 建立作品空间

- [ ] 打开项目管理页
- [ ] 新建项目
- [ ] 选择空白项目或文本上传项目
- [ ] 补充项目简介
- [ ] 选择画风预设或填写风格名
- [ ] 选择导演手册预设或填写导演手册

Expected outcome:

- 项目不是一个空壳名字，而是具备基础生产设定

### 02. 建立章节结构

- [ ] 进入章节工作台
- [ ] 导入或补齐章节目录
- [ ] 对当前章节执行：
  - 手动绑定原文章节
  - 或智能绑定本章
- [ ] 必要时对全项目执行自动绑定未映射章节

Expected outcome:

- 每个章节都有明确原文来源，不再直接消费整本书上下文

### 03. 建立镜头板

- [ ] 当前章节新建镜头
- [ ] 填写标题和摘要
- [ ] 使用筛选和搜索检查镜头列表
- [ ] 必要时执行镜头排序
- [ ] 必要时批量修改镜头状态

Expected outcome:

- 镜头成为真实生产单元，而不是零散备注

### 04. 建立首轮视觉结果

- [ ] 选择当前镜头
- [ ] 检查自动生成的 prompt
- [ ] 执行镜头出图
- [ ] 在候选图里选择结果
- [ ] 提升为场景物料

Expected outcome:

- 当前镜头不只是“有图”，而是能沉淀成可复用的共享资源

### 05. 做资源管理

- [ ] 检查当前镜头绑定的场景资源
- [ ] 确认该资源来源于当前章节还是其他章节
- [ ] 查看该资源影响了哪些镜头
- [ ] 查看是否有旧版本引用尚未同步

Expected outcome:

- 资源升级不再是黑箱，用户可以看到影响范围

### 06. 做镜头审阅

- [ ] 当前镜头结果可用时点“审核通过”
- [ ] 当前镜头结果不可用时点“打回返工”
- [ ] 对一组镜头批量标记待处理 / 处理中 / 返工

Expected outcome:

- 镜头状态不只是生产状态，也是审阅状态

### 07. 做流程链配置

- [ ] 进入章节流程或镜头流程
- [ ] 在 Studio 中编辑执行链
- [ ] 保存
- [ ] 确认顶部宿主 badge 是否正确

Expected outcome:

- Flow 不再是项目级总平面，而是明确附着在章节或镜头上的执行链

## Current Web Hint Requirements

以下暗示和引导必须始终存在：

- [ ] `workspace` 顶部有“继续生产”主卡
- [ ] `workspace` 中有“推荐上手路径”清单
- [ ] 章节页有“下一步建议”卡片
- [ ] 章节页有“本章操作路径”卡片
- [ ] 镜头区有批量管理提示
- [ ] 镜头编辑区有明确审阅动作
- [ ] Studio 顶部有当前宿主 badge

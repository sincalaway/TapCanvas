# 纳米漫剧流水线镜头列表行线框

目标：定义 `episodes/:episodeId/storyboard` 左侧镜头列表的单行结构、信息优先级和快捷操作，避免镜头列表变成低信息密度的长文案堆叠。

## 1. 页面目标

镜头列表行要解决四件事：

1. 快速定位当前镜头
2. 快速判断是否有风险
3. 快速看出生产/审核卡在哪
4. 快速进入重绘、候选、审核动作

## 2. 单行结构

建议每行固定为三段：

```text
┌─────────────────────────────────────┐
│ 左：编号/选中态/缩略图              │
│ 中：标题/摘要/状态/风险             │
│ 右：更新时间/快捷操作               │
└─────────────────────────────────────┘
```

## 3. 行内容建议

### 3.1 左侧

- 当前选中态
- 镜头编号
- 小缩略图

如果没有图：

- 用占位缩略块，不要空白

### 3.2 中间主信息

优先展示：

- 标题
- 一句话摘要
- 生产状态 badge
- 审核状态 badge
- 风险状态 badge

次级展示：

- 角色数量
- 场景名
- 下游视频数

### 3.3 右侧

- 最近更新时间
- 快捷操作按钮

建议按钮：

- `预览`
- `重绘`
- `候选`
- `审核`

## 4. 行示意

```text
[023] [thumb]
宫殿正反打
皇帝与侍卫对视，气氛压迫
[候选已出] [待审核] [结果过期]
场景：太和殿  下游：2 个视频
10:32   [预览] [重绘] [审核]
```

## 5. 信息优先级

从高到低：

1. 当前选中态
2. 风险状态
3. 标题
4. 审核状态
5. 生产状态
6. 更新时间

原因：

- 用户在镜头列表里最需要先看到“哪条有问题”

## 6. 选中态规则

### 6.1 当前镜头

- 左边框高亮
- 背景轻微区分
- 标题加粗

### 6.2 有风险但未选中

- 用风险色细边或角标
- 不要盖过当前选中态

## 7. 状态显示规则

### 7.1 Badge 数量限制

每行最多 3 个状态 badge：

- 生产状态 1 个
- 审核状态 1 个
- 风险状态 1 个

### 7.2 风险优先

如果空间不足：

- 保留风险 badge
- 生产/审核状态可折叠成更短文案

## 8. 缩略图规则

### 8.1 有图时

- 固定比例
- 不要大图，列表重在扫描

### 8.2 无图时

- 显示状态占位
- 可在占位中写 `未生成` / `无候选`

## 9. 快捷操作规则

### 9.1 默认悬停显示

为了密度，部分按钮建议 hover 时出现。

### 9.2 常驻按钮

常驻只留：

- `预览`
- `审核`

### 9.3 二级按钮

悬停或展开时再展示：

- `重绘`
- `切候选`
- `查看影响`

## 10. 列表分组建议

支持按 scene 分组：

- `Scene 1`
- `Scene 2`

每个 scene 头部可以显示：

- 镜头数
- 风险数
- 已通过数

## 11. 批量选择

如果未来支持批量操作：

- 行首加 checkbox
- 批量模式与普通浏览模式分开
- 不要默认一直显示一排 checkbox

## 12. 空状态

### 12.1 没有镜头

- 提示先生成分镜
- 给出创建入口

### 12.2 当前筛选无结果

- 提示修改筛选条件

## 13. 对当前仓库的落地建议

建议最终拆成：

- `ShotList`
- `ShotListSectionHeader`
- `ShotListRow`
- `ShotListRowActions`

如果接到 `apps/web`：

- 左侧列表组件只负责导航和状态展示
- 详情逻辑全部交给右侧栏

## 14. 首版 MVP

首版最小实现：

1. 缩略图
2. 标题
3. 一句话摘要
4. 三类状态 badge
5. 最近更新时间
6. 预览 / 审核快捷操作

先不做：

- 拖拽排序
- 自定义列
- 行内评论展开

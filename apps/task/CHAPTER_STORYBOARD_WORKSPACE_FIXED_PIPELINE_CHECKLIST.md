# 章节分镜工作台固定流水线 Checklist

更新时间：2026-04-03  
状态：已完成需求定义  
范围：`apps/web`、`apps/hono-api`、`apps/agents-cli`、`project-data/books/*/index.json`

目标：把“上传小说文本后，按章节完成漫剧创作”的当前通用对话/画布能力，收敛为一个可重复、可追溯、可稳定续跑的章节生产工作台。  
定位：这不是“给画布换个皮肤”，而是把章节漫剧生产固化成**固定流程工作台**。  
核心原则：**先通读正文 -> 再抽资产 -> 再预生产 -> 再逐镜头 JSON -> 再分镜图生成**。  
核心红线：**agent 稳定性优先，显式失败优先，零隐式回退，零语义硬编码补丁。**

---

## 0. 最终结论

- [x] 章节工作台是**章节漫剧生产的主入口**
- [x] 通用画布继续保留，但降级为**底层执行图与高级调试入口**
- [x] 用户在工作台里不再“自己搭流程”，而是操作固定 SOP
- [x] SOP 不允许跳过“预生产资产”直接宣称章节完成
- [x] 每个镜头都必须能追溯到：
  - [x] 章节正文证据
  - [x] 角色/场景/道具/风格参考
  - [x] `tapcanvas-storyboard-expert` 生成的 JSON 提示词
  - [x] 分镜图结果
  - [x] 运行日志与错误原因

---

## 1. 产品边界

### 1.1 本次要解决的问题

- [x] 上传长文本后，系统已经明确书和章节，但当前交互仍要求用户理解节点、连线、素材依赖和执行顺序
- [x] 章节生产缺少固定主界面，导致“正文、资产、分镜、提示词、执行状态”分散在聊天和画布里
- [x] 章节前置资产缺口经常被跳过，造成：
  - [x] 场景图漂风格
  - [x] 角色状态不连续
  - [x] 道具缺失
  - [x] 只生成文本节点却误判完成
- [x] 镜头 JSON 提示词与生成结果之间缺乏稳定映射，难以单镜头返工
- [x] agent 当前虽可生成内容，但缺少一个**稳定的章节级流程承载面**

### 1.2 本次不解决的问题

- [x] 首版不引入复杂多用户协同编辑
- [x] 首版不做完整视频剪辑器能力
- [x] 首版不做企业级排期/看板/工单系统
- [x] 首版不做自由流程设计器增强
- [x] 工作台不做成新的独立站点壳；仍基于当前项目内打开，但打开形式是**覆盖当前项目的全屏弹窗层**，不是浮动面板

## 2. 产品定义

### 2.1 工作台定义

- [x] 能力名称确定为：`章节分镜工作台` / `Chapter Storyboard Workspace`
- [x] 工作台是**章节级固定生产界面**
- [x] 工作台服务的主任务等价于：
  - [x] “完成第 X 章节的独立分镜创作”
  - [x] “先通读章节全文”
  - [x] “补齐预生产资产”
  - [x] “每个镜头的提示词用分镜图 skill 生成 JSON 结构提示词”
  - [x] “每个镜头产出独立分镜图节点”

### 2.2 用户心智模型

- [x] 主对象是 `章节`
- [x] 章节下有 `资产层`
- [x] 章节下有 `镜头层`
- [x] 章节下有 `执行层`
- [x] 用户平时操作的是“章节工作台”，不是“自由 Flow”

## 3. 页面结构

### 3.1 固定布局

- [x] 左侧：章节资产栏
- [x] 中间：当前片段脚本编辑区
- [x] 下方：章节分镜条 / 镜头时间线
- [x] 右上：章节控制区
- [x] 右侧：当前镜头预览 / 结果区

### 3.2 布局目标

- [x] 用户在同一页面能同时看到：
  - [x] 本章缺什么资产
  - [x] 当前已拆多少镜头
  - [x] 当前片段脚本与引用资产
  - [x] 当前镜头图或视频预览是否成功
- [x] 不允许重要信息被折叠到聊天记录深处
- [x] 不允许把工作台做成小型浮动面板；其信息密度和交互深度要求决定它必须是全屏弹窗
- [x] 本章正文全文不是默认主编辑面，而是作为“章节证据源”通过抽屉、切换卡或引用面板进入

## 5. 固定 SOP

### 5.1 固定阶段

- [x] 阶段 1：读取章节全文
- [x] 阶段 2：抽取章节实体
  - [x] 角色
  - [x] 场景
  - [x] 道具
  - [x] 节拍
- [x] 阶段 3：检查章节连续性锚点
- [x] 阶段 4：补齐预生产资产
- [x] 阶段 5：生成章节分镜脚本
- [x] 阶段 6：逐镜头调用 `tapcanvas-storyboard-expert` 生成 JSON
- [x] 阶段 7：逐镜头生成分镜图
- [x] 阶段 8：章节验收

### 5.2 固定规则

- [x] 只要 `required` 资产缺失，不允许章节进入完成态
- [x] 只要镜头 JSON 未完成，不允许镜头进入“已出图完成”
- [x] 只要镜头图缺失，不允许章节被标成 `storyboard completed`
- [x] `继续当前章节未完成步骤` 必须从最近未完成 stage 接着跑，不允许从头静默重来

## 8. Agent 智能体稳定性（P0）

### 8.1 主原则

- [x] 主代理和子代理必须服务于固定章节 SOP，而不是每轮自由发挥
- [x] agent 的自由度只存在于“如何拆镜头 / 如何组织提示词 / 如何利用已有资产”
- [x] agent 没有权力跳过工作台定义的阶段门禁

### 8.2 结构化输入要求

- [x] 每次章节运行必须拿到稳定的作用域输入：
  - [x] `projectId`
  - [x] `bookId`
  - [x] `chapterId`
  - [x] 当前章节正文
  - [x] 章节实体与连续性注入
  - [x] 现有资产事实
  - [x] 上一章尾帧
  - [x] `styleBible`
- [x] 缺任一关键字段时显式失败

### 8.3 结构化输出要求

- [x] agent 最终必须返回结构化章节结果，而不是只返回一段文案
- [x] 结果至少包含：
  - [x] 章节状态摘要
  - [x] 资产缺口列表
  - [x] 镜头列表
  - [x] 每镜头 JSON
  - [x] 每镜头引用资产
  - [x] 当前阶段是否完成
  - [x] 下一步动作

### 8.4 稳定性红线

- [x] 禁止 agent 因上下文不足自动降级为“仅文本建议”却不暴露失败
- [x] 禁止 agent 把“已创建节点”当成“已生成资产”
- [x] 禁止 agent 把“有 prompt”当成“有镜头图”
- [x] 禁止 agent 把“有章节标题”当成“已通读章节正文”
- [x] 禁止 agent 静默忽略缺失的风格锚点
- [x] 禁止 agent 在 chapter-grounded 任务里生成无参考绑定的图片节点
- [x] 禁止 agent 使用本地关键词补丁代替语义判断

### 8.5 agent 运行时门禁

- [x] 门禁 A：未拿到章节正文，不允许进入镜头拆解
- [x] 门禁 B：未完成章节实体抽取，不允许进入预生产
- [x] 门禁 C：`required` 资产缺失，不允许进入镜头出图完成态
- [x] 门禁 D：镜头 JSON 缺失，不允许调用镜头出图
- [x] 门禁 E：镜头图缺失，不允许章节验收 satisfied

### 8.6 agent 自检

- [x] 每轮章节生成完成前，agent 必须执行自检：
  - [x] 本轮是否真的读取了章节正文
  - [x] 本轮是否列出缺失资产
  - [x] 本轮是否先补了 `required` 资产
  - [x] 每个镜头是否有 JSON
  - [x] 每个镜头是否有图
  - [x] 当前回合能否真的被标记为章节完成
- [x] 自检失败时必须继续修正或显式失败

### 8.7 skill 使用稳定性

- [x] 逐镜头 JSON 提示词必须由 `tapcanvas-storyboard-expert` 负责
- [x] 章节流程编排必须使用统一章节工作流 skill，不允许每轮临时变体
- [x] skill 只承载方法论，不承载 case-specific 补丁
- [x] 若 skill 未加载成功，必须显式失败并记录技能缺失

## 9. 参考图与风格稳定性（P0）

### 9.1 统一风格锚点规则

- [x] 所有 chapter-grounded 图片生成都必须有参考图
- [x] 参考图来源优先级：
  - [x] 当前镜头显式资产绑定
  - [x] 当前章节资产绑定
  - [x] `styleBible.referenceImages`
  - [x] 上一章尾帧 continuity
- [x] 不允许场景图裸奔生成

### 9.2 持久化规则

- [x] 参考图必须持久化在节点字段或真实连边中
- [x] 不允许只在 prompt 文案中口头声明“延续上章风格”
- [x] 每个镜头必须能在详情区看到其参考资产来源

### 9.3 失败策略

- [x] 存在风格锚点但节点未持久化参考绑定时，回合必须报错或至少高优先级诊断
- [x] style bible 缺失且章节要求统一风格时，必须显式显示 `style_anchor_missing`

## 10. 章节资产抽取标准（能抽尽抽）

### 10.1 角色

- [x] 本章出现且后续会复用的角色必须抽出资产
- [x] 角色资产最低要求：
  - [x] 角色卡
  - [x] 状态锚点
  - [x] 三视图

### 10.2 场景

- [x] 本章反复出现或对空间关系重要的场景必须抽出
- [x] 场景资产最低要求：
  - [x] 主空间参考图
  - [x] 时段/光照说明
  - [x] 材质/空间结构说明

### 10.3 道具

- [x] 对剧情推进、识别性或后续复用重要的道具必须抽出
- [x] 不靠关键词硬编码“春秋蝉”之类个案
- [x] 判断标准必须基于章节实体抽取 + 复用价值 + 视觉关键性

### 10.4 风格锚点

- [x] 项目级 / 书级 style bible 必须作为正式资产源
- [x] 风格锚点必须能被镜头和场景资产复用

### 11.1 主从关系

### 11.2 映射要求

- [x] 镜头卡可定位到节点

## 17. MVP 验收标准

- [x] 用户进入章节工作台后，2 秒内看到本章正文、资产列表、镜头列表、章节状态
- [x] 点击“继续当前章节未完成步骤”后，系统按固定 SOP 推进，而不是自由发挥
- [x] 缺 `required` 资产时，章节不能被标记为 completed
- [x] 每个镜头都能看到 JSON 提示词
- [x] 每个镜头都能看到引用的参考资产
- [x] 每个镜头都能看到分镜图或明确失败原因
- [x] chapter-grounded 场景图、角色图、道具图都必须有参考绑定
- [x] agent 回合不能再出现“只有文本/只有 patch 节点但误判章节完成”的假阳性

## 18. 最终产品判断标准

- [x] 用户上传文本后，不需要理解节点协议、不需要自己判断先做资产还是先做镜头，只需要打开某一章节工作台，就能稳定完成该章节的独立分镜创作，并且每个镜头都可追溯、可重跑、可诊断。

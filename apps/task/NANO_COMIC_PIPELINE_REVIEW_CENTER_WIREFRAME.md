# 纳米漫剧流水线审核中心线框

目标：定义跨项目审核中心的列表结构、筛选方式、批量操作和详情联动，避免后续把审核中心做成一张无重点的大表。

## 1. 页面目标

审核中心只做三件事：

1. 收敛待审核对象
2. 让审核人快速判定优先级
3. 支持批量处理和进入详情

不做：

- 不承担生产编辑
- 不承担复杂资产管理
- 不承担项目配置

## 2. 页面结构

建议页面分为 3 区：

```text
┌─────────────────────────────────────────────┐
│ 顶部统计 + 快捷筛选                         │
├───────────────┬─────────────────────────────┤
│ 左侧筛选面板   │ 中间审核列表                │
│               │                             │
│               │                             │
├───────────────┴─────────────────────────────┤
│ 底部批量操作栏 / 空状态 / 分页             │
└─────────────────────────────────────────────┘
```

## 3. 顶部统计区

顶部第一屏必须先告诉用户“哪里最堵”。

建议卡片：

- `待审核总数`
- `导演确认数`
- `高风险数`
- `返工中数`
- `已阻塞下游数`

每张卡片支持点击筛选。

## 4. 左侧筛选面板

### 4.1 必须有的筛选项

- 项目
- 剧集
- 对象类型
- 审核状态
- 风险级别
- 是否阻塞下游
- 是否需要导演确认
- 当前责任人
- 当前审核人
- 更新时间范围

### 4.2 对象类型

- `asset`
- `shot`
- `video_segment`

### 4.3 风险级别

- `normal`
- `warning`
- `stale`
- `blocked`

### 4.4 筛选交互原则

- 默认保存上次筛选状态
- 支持一键恢复默认
- 常用筛选提供快捷 chips

## 5. 审核列表

### 5.1 列表推荐结构

每一行不是单纯表格文本，建议做成“信息密集卡片行”。

字段建议：

- 选择框
- 风险徽标
- 对象类型
- 标题
- 项目 / 剧集
- 当前状态
- 当前责任人
- 最近审核人
- 下游影响数量
- 更新时间
- 快捷操作

### 5.2 卡片行示意

```text
[ ] [阻塞] [镜头] 第 08 集 / 镜头 023 / 宫殿正反打
项目：王朝疑云   剧集：E08   责任人：张三   审核人：李四
风险：角色主设已更新，下游 2 个视频需复检
更新时间：今天 10:32
[查看详情] [通过] [打回] [驳回]
```

### 5.3 信息优先级

优先级从高到低：

1. 阻塞级风险
2. 是否导演确认
3. 对象标题与类型
4. 项目 / 剧集
5. 当前责任人
6. 更新时间

## 6. 快捷操作

每行允许有限快捷操作：

- `查看详情`
- `通过`
- `打回修改`
- `驳回重做`
- `指派`

规则：

- 快捷操作只保留高频动作
- 复杂配置进入详情侧栏

## 7. 批量操作栏

当列表选中项 > 0 时出现底部批量栏。

支持：

- 批量通过
- 批量打回修改
- 批量标记导演确认
- 批量重新指派
- 批量查看影响对象

批量操作前必须校验：

- 对象类型是否一致
- 当前用户是否有权限
- 是否存在阻塞冲突

## 8. 详情联动

点击 `查看详情` 后，建议右侧抽屉或新页打开统一审核面板。

详情区应复用：

- [Review Panel Wireframe](./NANO_COMIC_PIPELINE_REVIEW_PANEL_WIREFRAME.md)

不要在审核中心列表里直接塞评论全文或巨大表单。

## 9. 列表状态

### 9.1 默认排序

建议默认：

1. `blocked`
2. `director_confirmation_required`
3. `updated_at desc`

### 9.2 空状态

区分三种：

- 没有任何待审核对象
- 当前筛选条件下无结果
- 当前用户无审核权限

### 9.3 加载状态

加载时优先展示骨架屏，不要整页空白。

## 10. 颜色与徽标建议

### 10.1 状态色

- `approved` -> 绿色
- `pending_review` -> 蓝色
- `changes_requested` -> 橙色
- `rejected` -> 红色

### 10.2 风险色

- `warning` -> 黄色
- `stale` -> 橙红
- `blocked` -> 深红

### 10.3 特殊标记

- `导演确认` -> 紫色
- `返工中` -> 青色

## 11. MVP 版本

首版审核中心最小实现：

1. 顶部统计卡片
2. 左侧筛选
3. 中间列表
4. 每行快捷操作
5. 批量操作栏
6. 详情联动到审核面板

首版先不做：

- 多视图切换
- 自定义列配置
- 高级保存筛选器

## 12. 对当前仓库的落地建议

建议最终落到：

- `apps/web/src/ui/...` 的审核中心页面
- 可复用现有 `stats` 面板的列表密度和右侧详情结构
- 详情内容复用统一审核面板组件

# 纳米漫剧流水线审核面板线框

目标：先定义审核面板的结构、信息优先级和关键按钮，避免 UI 先做成“评论框 + 两个按钮”的空壳。

相关文档：

- [Sitemap](./NANO_COMIC_PIPELINE_SITEMAP.md)
- [Role Matrix](./NANO_COMIC_PIPELINE_ROLE_MATRIX.md)
- [Review Flow](./NANO_COMIC_PIPELINE_REVIEW_FLOW.md)
- [Dependency Rules](./NANO_COMIC_PIPELINE_DEPENDENCY_RULES.md)

## 1. 适用页面

这套审核面板建议复用于：

1. 资产详情页
2. 镜头详情页
3. 视频片段详情页

差异只在中间内容，不在结构骨架。

## 2. 面板布局

建议固定为右侧侧栏，宽度稳定，不做漂浮弹窗优先。

```text
┌──────────────────────────────┐
│ 审核状态 + 责任人 + 时间     │
├──────────────────────────────┤
│ 风险摘要 / 下游影响          │
├──────────────────────────────┤
│ 审核操作区                   │
├──────────────────────────────┤
│ 驳回原因 / 返工配置          │
├──────────────────────────────┤
│ 评论线程                     │
├──────────────────────────────┤
│ 审核历史时间线               │
└──────────────────────────────┘
```

## 3. 顶部状态区

必须展示：

- 当前审核状态
- 当前责任人
- 最近审核人
- 最近审核时间
- 是否阻塞下游
- 是否需要导演确认

示例字段：

- `状态: 待审核`
- `责任人: 张三`
- `最近审核: 李四 · 10:32`
- `下游影响: 3 个视频片段`

## 4. 风险摘要区

作用：让用户先看到“为什么现在要处理”。

内容建议：

- 一句话风险摘要
- 风险等级
- 受影响下游对象数量
- 推荐动作

示例：

- `角色主设已更新，当前镜头结果过期`
- `下游 2 个视频片段需复检`
- `建议先重绘，再重新提交审核`

## 5. 审核操作区

这是核心操作区。

### 5.1 基础按钮

- `提交审核`
- `通过`
- `打回修改`
- `驳回重做`

### 5.2 辅助按钮

- `指派返工`
- `标记导演确认`
- `查看影响对象`
- `批量处理下游`

### 5.3 按角色控制

- `artist/editor` 默认看到 `提交审核`
- `reviewer` 默认看到 `通过 / 打回修改 / 驳回重做`
- `director` 额外看到 `导演确认`

## 6. 驳回配置区

只有点击 `打回修改` 或 `驳回重做` 后展开。

字段建议：

- 一级原因
- 二级原因
- 文字说明
- 是否阻塞下游
- 返工责任人
- 截止时间

示例布局：

```text
[原因一级 v]
[原因二级 v]
[补充说明 textarea]
[x] 阻塞下游继续生产
[返工责任人 v]
[截止时间]
[确认驳回]
```

## 7. 评论线程区

评论区不能喧宾夺主，但必须常驻。

建议能力：

- 最新评论优先
- 支持 `@成员`
- 支持“带审核动作的评论”
- 支持引用当前对象版本

评论输入区建议包含：

- 文本框
- `@`
- 快捷插入“建议动作”

## 8. 审核历史区

用时间线展示，不要只是表格。

节点至少包括：

- 谁提交了审核
- 谁通过/驳回
- 驳回原因
- 是否指派返工
- 是否重新提交
- 最终是否通过

## 9. 三类对象的差异

### 9.1 资产审核面板

额外展示：

- 被引用剧集数
- 被引用镜头数
- 当前版本号

### 9.2 镜头审核面板

额外展示：

- 上一镜 / 下一镜状态
- 连续性告警
- 候选结果数

### 9.3 视频审核面板

额外展示：

- 来源镜头范围
- 节奏风险
- 是否可导出

## 10. 空状态与错误状态

### 10.1 空状态

如果还没提交审核：

- 提示当前还未进入审核
- 给出下一步动作按钮

### 10.2 错误状态

如果缺少上游上下文：

- 显式展示缺什么
- 不允许假装可审核

## 11. 移动端与窄屏

窄屏下不要全塞在一个抽屉里。

建议：

- 默认只展示状态和操作区
- 评论和历史折叠成页签

## 12. 首版 MVP 面板

首版最小化建议：

1. 顶部状态区
2. 审核操作区
3. 驳回配置区
4. 评论线程
5. 历史时间线

先不做：

- 多级审批树
- 高级统计图
- 超复杂过滤器

## 13. UI 落地建议

如果接下来开始做页面，优先落在：

- `episodes/:episodeId/storyboard` 右侧栏
- `episodes/:episodeId/review` 列表 + 详情布局
- `review-center/pending` 列表行快捷审核

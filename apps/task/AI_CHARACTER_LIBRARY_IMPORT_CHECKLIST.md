# AI Character Library Import Checklist

## Goal

- [x] 明确本次目标是“抓取 TapNow 角色库并持久化到我们自己的数据库/资产表”，不是前端运行时代理外部库
- [x] 前端角色库页面仅消费本地持久化数据
- [x] “应用到画布”基于本地持久化记录工作，不再直接请求 TapNow 角色列表接口

## Storage

- [x] 复用现有 `assets` 持久化模型，确认无需新增高成本迁移
- [x] 设计角色库记录的数据结构：基础属性、筛选字段、四张图 URL、来源标记、同步时间
- [x] 设计导入状态记录：最后同步时间、总数、分页进度或统计摘要

## Backend

- [x] 新增后端导入入口：按用户触发从 TapNow 拉取角色库数据
- [x] 导入逻辑支持分页抓取，不只抓第一页
- [x] 导入逻辑支持幂等更新，避免重复插入同一角色
- [x] 新增后端本地查询入口：读取已导入角色库列表与筛选条件
- [x] 保留必要的图片访问方案，保证前端能稳定展示已导入记录

## Frontend

- [x] 角色库弹窗改为读取本地查询接口

- [x] 列表、筛选、详情、应用到画布四步链路基于本地数据闭环
- [x] 清理或停用上一版运行时外部列表代理依赖

# TapCanvas Signature Pipeline Alignment Checklist

## Product Definition

- [x] 明确最终形态不是“项目管理工具”，而是“把文本作品按章节稳定生产为连续内容的导演系统”。
- [x] 明确核心链路：创建项目 -> 上传文本并锁定画风 -> 按章节推进 -> 项目级资产探知/复用 -> 流程可回溯 -> 连续稳定高质量输出。
- [ ] 将这条链路固化为全站一级叙事，而不是分散在多个页面说明中。
- [x] 将这条链路固化为全站一级叙事，而不是分散在多个页面说明中。

## Phase 1: Production Kickoff

- [x] `Workspace` 首页给出“今天先做什么”的唯一主任务入口。
- [x] 新建项目入口保留“上传文本创建项目”为默认起手式，不再鼓励空白项目。
- [x] 将项目创建弹窗重构为“项目启动向导”，强调作品、文本、画风、导演规则四个启动锚点。
- [x] 创建成功后，若章节尚未就绪，提供稳定的“导入中/等待进入第一章”过渡态，而不是让路由失败。
- [x] 把“创建后会发生什么”明确写给用户：拆书、章节进入、资产沉淀、后续推进方式。

## Phase 2: Chapter Production Console

- [x] 章节页新增“本章现在该做什么”阶段卡。
- [x] 章节页新增直达锚点：文本导入、章节绑定、镜头板、镜头生产。
- [x] 将章节页进一步收敛为“逐章推进控制台”，下沉弱相关信息。
- [x] 把章节状态表达从技术状态改成生产状态，围绕推进、待审、返工、归档展开。
- [x] 强化“先校准文本窗口，再建镜头，再出图”的顺序约束。

## Phase 3: Shared Project Memory

- [x] 把项目级资产从“资源模块”升级成“项目共享记忆”。
- [ ] 明确角色、场景、道具、画风锚点都属于项目层可探知资产。
- [x] 明确角色、场景、道具、画风锚点都属于项目层可探知资产。
- [x] 在章节内更清楚地显示“当前镜头用了哪些共享资产、来自哪里、影响哪些镜头”。
- [x] 让资产提升/复用成为章节生产中的显性动作，而不是隐藏在结果处理后。

## Phase 4: Traceability

- [x] 为项目/章节/镜头提供可回看的生产履历：创建、绑定、生成、选图、返工、提升资产。
- [x] 将流程记录从技术日志提升为产品可读的时间线。
- [x] 让用户能快速定位“这一章为什么现在长这样”。

## Phase 5: Quality and Continuity

- [x] 将“连续稳定高质量输出”落实成界面约束，而不是仅靠 prompt 文案。
- [x] 增加更明确的连续性提醒：角色一致、场景一致、风格一致、章节上下文一致。
- [ ] 为返工提供质量原因定位，而不是只给失败状态。
- [x] 为返工提供质量原因定位，而不是只给失败状态。

## Current Focus

- [x] 收拢首页与章节页的主任务感知。
- [x] 收拢项目创建后的第一跳稳定性。
- [x] 收拢“章节尚未就绪时”的用户等待体验。
- [x] 收拢项目级资产与流程回溯的产品表达。

# TapCanvas Semantic Analysis Protocol

## 目标

明确区分：

- 结构提取
- 语义理解

结构提取可以由脚本完成；语义理解必须由 agent 基于证据完成。

## 一、两层职责

### 结构层

输入：

- `assets/demo/*.json`

输出：

- 节点统计
- 连线统计
- prompt 样本
- 模式命中
- 候选 sceneType / action

结构层只回答：

- 这里面有什么
- 高频出现了什么模式
- 哪些动作可能相关

结构层不能回答：

- 创作意图是什么
- 为什么这个 workflow 成功
- 哪些案例细节可以迁移
- 当前用户任务与该案例是否语义同类

### 语义层

输入：

- 结构层摘要
- 少量必要的原始证据
- 当前用户任务

输出：

- 创作意图判断
- 可迁移规律
- 场景类型判断
- 当前状态判断
- 最合适的下一步动作

## 二、推荐流程

1. 先读索引层
2. 若无 `analysis.json`，生成结构层摘要
3. agent 基于结构层摘要做语义分析
4. 只在证据不足时，选择性回看原始 JSON
5. 最终如需沉淀长期知识，再写 `analysis.json`

## 三、禁止事项

- 不允许把脚本输出直接当成长期语义知识
- 不允许把模式命中直接等同于创作意图
- 不允许用正则代替语义理解
- 不允许省略语义分析步骤就更新长期知识资产

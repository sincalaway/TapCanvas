# TapCanvas Style Adaptation Patterns

## 目标

把“同一镜头跨风格改编”的经验拆成可复用模式，供 agents-cli 在 AI 创作中优先调用。

适用范围：

- 动漫截图改真人写实
- 动漫截图改 AAA 游戏 CG
- 动漫截图改国漫 3D
- 已有稳定关键帧的动作视频化
- IP 改编、预告片镜头预演、跨媒介视觉开发

对应案例：`assets/demo/003.json`
结构化分析：`ai-metadata/workflow-patterns/003.analysis.json`

## 一、最核心的方法

### 1. 先锁镜头语义，再换风格域

先把以下内容锁住：

- 构图
- 景别
- 主体比例
- 朝向与站位
- 动作节拍
- 空间关系
- 光源方向、强度、色温与对比

不要一开始就同时追求：

- 风格转换
- 角色修脸
- 天气变化
- 道具替换
- 视频动作

先保证“这是同一个镜头”，再讨论“它看起来像什么媒介”。

### 2. 风格转换只改渲染，不改事实

把以下目标理解为“渲染域切换”：

- 真人写实
- 日系 AAA 游戏 CG
- 国漫 3D
- 线稿中间态

这一步只允许改变：

- 材质表达
- 贴图细节
- 渲染精度
- 视觉质感
- 风格化程度

不允许顺手改变：

- 构图与透视
- 主体身份
- 景别
- 光影逻辑
- 空间位置关系

### 3. 风格锁定后再做局部微调

局部微调必须后置。

推荐拆分：

1. 先做一次干净的风格域切换
2. 再单独改天气
3. 再单独改眼睛或面部细节
4. 再单独改护额、武器、符号或局部道具
5. 再决定是否拼接、换脸或进入视频

不要把这些需求塞进同一条 prompt。

### 4. 视频层只负责动作，不负责重建镜头

如果关键帧已经稳定，视频 prompt 只负责：

- 谁动
- 怎么动
- 往哪动
- 镜头是否固定或只允许一条路径
- 环境持续状态是什么
- 明确禁止什么

不要把视频层当成：

- 风格修复器
- 修脸器
- 重构图器
- 自动补镜头设计器

## 二、四种高频模式

## 模式 A：同镜头跨风格迁移

用途：把同一镜头迁移到另一种视觉媒介，同时保留叙事可读性。

模板：

```text
严格保留参考图的构图、景别、主体比例、朝向、动作语义与空间关系，
并以原图光线为唯一参考（保持光源方向、强度、色温与对比不变），
仅将画面转换为[目标风格域]。
不要新增或删减内容。
```

适合：

- 动漫转真人
- 动漫转 AAA 游戏 CG
- 动漫转国漫 3D
- 写实版与风格化版比稿

## 模式 B：光影逻辑冻结

用途：防止风格切换时把原镜头情绪与空间逻辑一起改掉。

模板：

```text
以原图光线为唯一参考，保持原始光源方向、强度、色温与对比不变，
不新增、不改动打光逻辑，只升级材质与渲染表现。
```

适合：

- 角色质感升级
- 动漫截图写实化
- 电影感强化但不想重设打光

## 模式 C：风格锁定后单变量微调

用途：在稳定风格版本上只改一个小变量。

模板：

```text
保持当前参考图其余所有内容不变，只修改[单个变量]。
保留镜头语义、角色身份、空间关系、整体风格与光影逻辑。
```

适合修改：

- 天气从晴天改多云
- 眼部状态变化
- 局部面部修正
- 护额、武器、符号、局部道具
- 双图拼接或局部换脸

不适合：

- 同时改风格、表情、动作、天气和镜头

## 模式 D：关键帧动作视频化

用途：把已锁定风格的静帧扩展成受控动作镜头。

模板：

```text
以该关键帧为首帧或尾帧，保持固定机位与当前景别不变。
只允许发生以下动作：[动作]
环境持续状态：[持续状态]
禁止：[禁止项]
```

推荐写法：

- 只允许发生以下动作：角色向左猛冲后瞬间停住
- 环境持续状态：狂风持续、烟尘持续、战场空间不变
- 禁止：不要新增镜头运动，不要新增多余手势，不要改变人物身份与服装

## 三、约束句式库

### 保持镜头语义

- 严格保留参考图的构图与景别
- 保持主体比例、朝向与空间关系不变
- 不改变透视、裁切、景别和镜头语义
- 只改渲染方式，不改画面事实

### 保持光影逻辑

- 以原图光线为唯一参考
- 保持原始光源方向、强度、色温与对比不变
- 不新增、不改动打光逻辑

### 风格转换约束

- 仅将画面转换为 [目标风格域]
- 保留原镜头的一切叙事事实
- 不要新增或删减内容

### 视频动作约束

- 强制固定机位
- 保持首帧画面景别拍摄
- 不能有任何镜头运动
- 只允许一个被批准的动作事件
- 不要加入 prompt 未提及的额外动作

## 四、推荐工作流

### 阶段 1：锁镜头

- 选择一个语义清楚的原始截图或关键帧
- 确认构图、主体、动作、空间关系可读

### 阶段 2：切风格域

- 只做一次干净的风格迁移
- 不混入局部修图与视频化需求

### 阶段 3：局部 polish

- 只改一个局部变量
- 每一步都保留镜头语义和风格统一性

### 阶段 4：视频化

- 只选择最稳定的关键帧
- 视频 prompt 只负责动作和禁止项
- 视频失败优先回到关键帧层修

## 五、失败信号

出现以下情况，说明 workflow 设计有问题：

- 风格转换后不再是同一个镜头
- 原图光影逻辑被顺手改写
- 同一步里混入多个变量修改
- 视频阶段开始承担修脸、换风格、重构图等任务
- 局部修图破坏了已锁定的角色身份或空间关系

## 六、使用建议

如果后续要让 agents-cli 使用这类知识，优先读取：

1. `assets/demo/index.json`
2. `ai-metadata/workflow-patterns/003.analysis.json`
3. 本文档

不要优先直接把 `assets/demo/003.json` 整份塞进上下文；原始工作流适合做证据来源，不适合直接做 prompt 规范。

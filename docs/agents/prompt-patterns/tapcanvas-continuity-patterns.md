# TapCanvas Continuity Patterns

## 目标

把成功工作流里的经验拆成可复用的创作模式，而不是复用案例本身。

适用范围：

- 多镜头分镜
- 强连续性的图生图迭代
- 稳定关键帧后的图转视频
- POV / 反打 / 小动作 suspense 场景

对应案例：`assets/demo/001.json`
结构化分析：`ai-metadata/workflow-patterns/001.analysis.json`

## 一、最核心的方法

### 1. 先锁世界，再做镜头

先生成一张“权威基底帧”，把以下信息锁住：

- 场景地点
- 主体身份
- 服装与道具
- 空间关系
- 色调和质感

不要在第一步就追求复杂动作。先把世界观和画面事实稳定下来。

### 2. 每次只改一个变量

连续性任务最常见的失败原因，是同一条 prompt 同时改太多东西。

推荐一次只改一项：

- 机位
- POV
- 道具状态
- 背景角色是否保留
- 主体朝向

如果要改多个变量，拆成多步图生图，而不是一步到位。

### 3. 把“不变项”写出来

不要假设模型会自动保留原图事实。

每次迭代都要显式写：

- 哪些内容必须不变
- 哪些角色必须保留
- 哪些空间关系必须不变
- 哪些内容不允许出现

### 4. 先图后动

视频生成不要承担“定世界 + 讲动作 + 修连续性”三种任务。

正确顺序是：

1. 先得到稳定关键帧
2. 再用关键帧生成视频
3. 视频 prompt 只负责动作、镜头和时间变化

## 二、四种高频模式

## 模式 A：基底帧锁定

用途：建立后续所有镜头共享的画面事实。

模板：

```text
[场景]，[主体]，[动作/道具]，[地点/时间]，[色调/质感目标]
```

示例骨架：

```text
画面是在[地点]，[主体身份与年龄]正在[动作]，画面保留[色调]，[纪录片/写实/低饱和]质感。
```

关键点：

- 先说清场景和主体
- 再补道具和动作
- 最后给视觉质感

## 模式 B：单变量连续性修改

用途：在保留参考图大部分内容的前提下，改一个关键变量。

模板：

```text
保持参考图其余所有内容完全不变，只修改[单个变量]。
保留[环境锚点]、[角色身份]、[空间关系]、[色调]。
```

适合修改：

- 机位从正面换到侧面
- 删掉一个背景乘客
- 让书本合上
- 让人物转头但环境不变

不适合：

- 同时改机位、情绪、灯光、动作、道具状态

## 模式 C：POV 镜头

用途：做第一人称镜头时避免穿帮。

模板：

```text
现在是[角色A]的第一人称 POV。
画面看到[角色B / 目标对象 / 目标动作]。
注意画面里不能出现[角色A]本人。
其余环境、构图和色调保持不变。
```

关键点：

- 必须写明 POV 属于谁
- 必须写明画面里不该出现谁
- 必须补一句环境与色调保持不变

## 模式 D：稳定关键帧转视频

用途：把已经稳定的静帧转换成有控制的视频。

模板：

```text
Generate video from image.
只允许发生以下动作：[动作]
镜头运动：[镜头运动]
环境持续状态：[持续状态]
禁止：[禁止项]
```

推荐写法：

- 只允许发生以下动作：角色抬起书本靠近镜头
- 镜头运动：轻微手持晃动，跟随车厢运动
- 环境持续状态：列车始终在行驶，背景始终在隧道中
- 禁止：不要翻页，不要新增人物动作，不要改变座位关系

## 三、约束句式库

这些句式的价值在于“限制漂移”，不是修辞。

### 保持不变

- 保持参考图其余所有内容完全不变
- 其他环境信息完全不变
- 构图、色调和角色关系保持不变
- 除了 [变量] 之外，其余内容全部保持不变

### 角色连续性

- 角色外貌保持一致
- 其他乘客依然存在
- 背景人物的样貌和服装与参考图一致

### POV 约束

- 注意这是第一人称 POV，所以看不见主角本身
- 镜头是 [角色] 的主视角，不要让镜头拥有者出现在画面中

### 视频约束

- 镜头始终轻微晃动
- 环境始终保持在 [地点]
- 不要增加额外肢体动作
- 不要让角色做 prompt 中未提及的行为

### 负面约束

- 不许出现任何文字
- 不要出现涂鸦
- 不要新增角色
- 不要改变空间位置关系

## 四、推荐工作流

### 阶段 1：基底帧

- 生成 1 张权威基底帧
- 确认人物、环境、色调、道具稳定

### 阶段 2：镜头扩展

- 从基底帧派生侧面、反打、POV 等镜头
- 每一步只改一个变量
- 对重要镜头做局部修正，直到连续性通过

### 阶段 3：视频化

- 只挑已经稳定的关键帧
- 视频 prompt 只负责运动
- 每个视频 prompt 都要同时包含：
  - 允许动作
  - 镜头运动
  - 环境持续状态
  - 明确禁止项

## 五、失败信号

出现以下情况，说明 prompt 设计有问题：

- 背景角色在镜头间变脸
- POV 镜头里看到了 POV 拥有者本人
- 视频里角色做了未要求的动作
- 环境从一个地点漂移到另一个地点
- 一条 prompt 同时改了三四件事情

## 六、使用建议

如果后续要让 agents-cli 使用这类知识，优先读取：

1. `assets/demo/index.json`
2. `ai-metadata/workflow-patterns/001.analysis.json`
3. 本文档

不要优先直接把 `assets/demo/001.json` 整份塞进上下文；原始工作流适合做证据来源，不适合直接做提示词规范。

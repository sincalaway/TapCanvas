# Canvas 组功能规范 v0（需求优先）

更新时间：2026-02-14  
适用范围：`apps/web/src/canvas/*`

## 1. 目标

本规范定义 TapCanvas 的“组（Group）”能力，以产品需求为准，不以历史实现为准。  
后续实现、重构、测试、评审均以本文件为唯一验收基线。

## 2. 领域模型

### 2.1 实体

- `Node`：画布节点。
- `GroupNode`：可容纳子节点的容器节点（`type === 'groupNode'`）。

### 2.2 唯一成员关系模型

- 子节点归属仅由 `parentId` 表达。
- 禁止并行维护第二套成员关系（如 `childNodeIds` 作为事实源）。
- 允许 `parentId = undefined`（顶层节点）。

### 2.3 不变量（必须始终成立）

1. 节点不得 `parentId === self.id`。
2. `parentId` 指向不存在节点时，必须在修复流程中被清空。
3. 非 `groupNode` 才允许作为子节点。
4. 组删除后，其子节点必须被显式处理（见 4.2）。
5. 序列化与反序列化后，`parentId` 关系保持一致。

## 3. 用户行为规范

### 3.1 打组（Group Selection）

输入：
- 选中 >= 2 个非组节点（同一层级或不同层级由产品策略决定，v0 先限制为同层级）。

行为：
1. 创建 `groupNode`。
2. 将被选中节点的 `parentId` 设为该组 ID。
3. 组尺寸自动包裹子节点（含 padding）。
4. 历史栈记录为单一步骤（可一次 undo）。

输出：
- 新建组处于选中状态；子节点默认取消选中。

### 3.2 解组（Ungroup）

输入：
- 选中一个或多个 `groupNode`。

行为：
1. 收集组内一层子节点（v0 不自动递归）。
2. 子节点 `parentId` 置空并转换为绝对坐标。
3. 删除组节点。
4. 历史栈记录为单一步骤。

输出：
- 解组后的子节点设为选中状态。

### 3.3 拖拽归组

输入：
- 拖拽结束事件（drag stop）。

行为：
1. 用命中规则判断应归属组（v0：中心点命中 + 最小面积优先）。
2. 归组变化时重写 `parentId` 与相对坐标。
3. 不触发多次 history 写入（一次拖拽只写一次）。

### 3.4 组尺寸变化

行为：
1. 组节点 resize 时可对子节点做比例缩放（v0 保留）。
2. 自动包裹逻辑与手动 resize 不互相抖动。
3. 非用户 resize（内部测量）不得触发子节点缩放。

### 3.5 聚焦模式（Focus Group）

行为：
1. 必须提供可达入口（右键菜单或快捷键或双击组标题）。
2. 面包屑展示当前焦点栈。
3. `Esc` / “上一级” / “退出聚焦”行为一致。

## 4. 关键边界策略

### 4.1 复制粘贴

- 粘贴组内节点时，若父组未同时粘贴，则 `parentId` 清空。
- 粘贴时若父组也复制，必须重映射到新组 ID。

### 4.2 删除策略

`removeSelected` 对组删除策略统一为：
- v0 默认：删除组时同时删除其一层子节点（级联删除）。
- 若未来改为“解散后保留子节点”，必须单独立项并更新本规范。

### 4.3 持久化策略（必须二选一）

当前仓库存在“存储时剥离 groupNode”的历史逻辑。v0 要求明确二选一并统一：

1. 保留组：`groupNode + parentId` 全量持久化（推荐）。
2. 禁用组：彻底移除组相关 UI 与逻辑。

当前选择：`方案 1（保留组）`。  
即：项目加载、导入、本地持久化、恢复路径都必须保留 `groupNode + parentId`。

## 5. 状态与接口契约

## 5.1 Store Actions（必须可调用且非空实现）

- `addGroupForSelection(name?)`
- `ungroupGroupNode(id)`
- `removeGroupById(id)`
- `renameGroup(id, name)`

### 5.2 快捷键契约

- `Mod+G`：打组
- `Mod+Shift+G`：解组
- `Esc`：有焦点组则退一级，无焦点则清空选择

快捷键定义与实际行为必须一致，不允许“常量存在但无实现”。

## 6. 验收测试（最小集）

1. 打组：2 个节点打组后，`parentId` 正确，undo/redo 正常。
2. 解组：解组后组节点消失，子节点回顶层且位置正确。
3. 拖拽跨组：节点从 A 组拖到 B 组后归属正确。
4. 组 resize：子节点位置/尺寸变换稳定，无抖动漂移。
5. 复制粘贴：父子映射正确，无悬挂 `parentId`。
6. 删除组：遵循 4.2 策略，边与历史栈正确。
7. 持久化：刷新后组结构不丢失（若选“保留组”方案）。

## 7. 与当前代码的差异（执行清单）

1. 将 `apps/web/src/canvas/store.ts` 中组 action 的 stub 实现补齐。
2. 统一校验逻辑到 `parentId`，移除 `childNodeIds` 事实依赖。
3. 修复 `KeyboardShortcuts`：实现 `Mod+G` / `Mod+Shift+G`。
4. 打通 `enterGroupFocus` 调用入口，或移除焦点模式死代码。
5. 持续确保所有加载/导入/恢复路径都走 `sanitizeGraphForCanvas`，避免引入旧分支。

## 8. 非目标（v0）

- 不支持多父组。
- 不支持组-组自动递归缩放策略（仅一层保障）。
- 不支持跨项目组模板复用。
